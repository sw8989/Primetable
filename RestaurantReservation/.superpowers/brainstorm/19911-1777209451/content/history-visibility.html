How visible should conversation history be?

The backend now stores threads — how much of this should the user actually see?

A

Silent persistence

Conversation is stored invisibly. The same thread continues across page reloads — the chat just picks up where it left off. No history panel, no thread list.

Simplest to build. User just notices their chat "remembers" them.

B

New chat + resume last

A "New conversation" button clears the chat. A "Resume" button (or auto-resume on load) brings back the last thread. No full history list.

One step up. User has control without a full sidebar.

C

Conversation history panel

A sidebar or dropdown lists past conversations (by date or restaurant context). User can click any to resume it. Like Claude.ai or ChatGPT sidebar.

Most powerful, most work. Makes threads a first-class feature.
